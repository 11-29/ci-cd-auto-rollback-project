CI/CD Pipeline with Automated Rollback

Built a CI/CD pipeline using GitHub Actions to automate deployment of a Dockerized Flask application on AWS EC2.

The pipeline builds the application, runs a test container, and performs a health check before promoting it to production.

Key Features:
✔ Automated deployment using GitHub Actions
✔ Docker-based container deployment
✔ Health check before production deployment
✔ Automatic rollback on failure

Failure Handling:
✔ Docker build failure stops deployment
✔ Container startup failure triggers rollback
✔ Health check failure restores previous version

Tech Stack:
Flask | Docker | GitHub Actions | AWS EC2 | Nginx
